# VV Hybrid Universe Public Positioning

This document defines how the public ecosystem should be read across the main repositories.

## Core Structure

- `VV Hybrid Universe` is the ecosystem
- `vv-hybrid-univers-manifesto` is the vision and philosophy layer
- `vv-hi-framework` is the invention and intellectual property layer
- `vv-nexus` is the system core and implementation center
- `vv-lea`, `vv-pulse`, `vv-team`, `VNEX`, and related surfaces are products built on top

## Public Reading Order

For someone discovering the ecosystem for the first time:

1. Read the manifesto to understand the worldview.
2. Read the framework repository to understand the invention layer.
3. Read `vv-nexus` to understand the system core and implementation context.
4. Read product repositories to understand what people can actually use.

## Why This Matters

Without this structure, the ecosystem can look like many disconnected experiments.

With this structure, the story becomes:

- there is a vision
- there is an original framework
- there is a system core
- there are products growing from that base

## Public Message

The simplest serious public explanation is:

`VV Hybrid Universe` is a local-first ecosystem for identity, memory, access and human-centered intelligence.

Its vision is declared in the manifesto.
Its invention layer is declared in `VV Hi Framework`.
Its implementation center is `VV Nexus`.
Its product surfaces include LEA, VNEX and VV Pulse.
